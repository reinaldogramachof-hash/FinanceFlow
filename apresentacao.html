01 — Introdução

O que é o FinanceFlow?

O FinanceFlow é uma aplicação web de gerenciamento financeiro pessoal desenvolvida como projeto acadêmico. O sistema permite ao usuário controlar receitas e despesas, organizar compromissos em uma agenda interativa, manter anotações pessoais e visualizar relatórios analíticos de sua vida financeira.

O sistema foi construído como uma SPA (Single-Page Application) — ou seja, toda a navegação ocorre sem recarregamento de página, proporcionando uma experiência fluída similar a aplicativos nativos. Adicionalmente, o sistema suporta PWA (Progressive Web App), permitindo que o usuário instale o FinanceFlow diretamente no celular ou desktop como um aplicativo.

Motivação: A maioria das pessoas não tem controle claro sobre para onde vai seu dinheiro. O FinanceFlow resolve isso centralizando receitas, despesas, agenda de pagamentos e análises em um único painel moderno e acessível.

02 — Tecnologias

Stack Tecnológico

O projeto foi desenvolvido com tecnologias modernas, open-source e gratuitas, divididas em três camadas: Frontend, Backend e Infraestrutura.

Frontend

📄
HTML5
Estrutura semântica do sistema. Uma única página (index.html) com todos os módulos, modais e elementos de navegação.
🎨
CSS3 + Variáveis CSS
Estilização completa com sistema de design baseado em :root variables. Tema escuro/claro sem JavaScript extra.
⚡
JavaScript ES2022
100% vanilla JS (sem frameworks). Módulos separados por funcionalidade usando async/await e Fetch API.
📊
Chart.js (CDN)
Biblioteca para geração de gráficos interativos: pizza, barras, linha e rosca. Carregado via CDN sem instalação.

Backend

🟢
Node.js ≥ 18
Runtime JavaScript no servidor. Executa as funções serverless na Vercel. Usa ES Modules (import/export).
🔌
Vercel Serverless Functions
Cada arquivo em /api/*.js vira automaticamente um endpoint HTTP. Sem necessidade de servidor dedicado.
🐘
PostgreSQL (Neon)
Banco de dados relacional hospedado gratuitamente no Neon. Conexão via pg (node-postgres) com pool de conexões.
🐘
PHP + SQLite (local)
Para desenvolvimento local, os mesmos endpoints existem em PHP com banco SQLite — sem necessidade de internet.

Infraestrutura e Deploy

▲
Vercel (Hosting)

🐙

📱
PWA (Service Worker)

04 — Banco de Dados

Modelagem de Dados

O banco de dados utilizado em produção é o PostgreSQL, hospedado gratuitamente no serviço Neon.tech. Para desenvolvimento local, o sistema usa SQLite com PHP/PDO. As tabelas são criadas automaticamente na primeira execução via CREATE TABLE IF NOT EXISTS.

Auto-criação de tabelas: Não é necessário executar nenhuma migration ou script SQL manualmente. O arquivo api/_db.js cria todas as tabelas na primeira requisição ao sistema.

Diferenciais do Banco de Dados

Recurso Implementação Benefício
Auto-criação CREATE TABLE IF NOT EXISTS Zero configuração manual

Filtro por mês TO_CHAR(date, 'YYYY-MM') = $1 Permite filtrar por mês sem converter no JS

COALESCE COALESCE(SUM(...), 0) Retorna 0 em vez de NULL quando não há dados
05 — API REST

Endpoints da API

Cada módulo possui seu próprio endpoint REST que responde a todos os métodos HTTP necessários. A API segue o padrão RESTful com retornos em JSON.

Transações — /api/transactions

MétodoParâmetrosRetorno
GET ?month=YYYY-MM &type= &category= { data: [...], summary: { income, expense } }
POST Body: { type, value, category, description, date } { id, message } — status 201
PUT ?id=N + mesmo body do POST { message } — status 200
DELETE ?id=N { message } — status 200

Agenda — /api/agenda

MétodoParâmetrosRetorno
GET ?month=YYYY-MM &status= Array de eventos ordenados por data/hora
POST Body: { title, date, time, type, status, description } { id, message } — status 201
PUT ?id=N + body completo { message }
DELETE ?id=N { message }

Anotações — /api/notes

MétodoParâmetrosRetorno
GET — Array de notas ordenadas por updated_at DESC
POST Body: { title, content, color } { id, message } — status 201
PUT ?id=N — atualiza updated_at = NOW() { message }
DELETE ?id=N { message }

Relatórios — /api/reports

MétodoParâmetrosRetorno
GET ?year=YYYY { monthly: [...], by_category: [...], totals: {...}, year }
06 — Funcionalidades

Módulos do Sistema

O FinanceFlow é composto por 5 módulos acessíveis pela barra lateral (desktop) ou barra de navegação inferior (mobile).

📊
Dashboard
Visão geral em tempo real das finanças do mês
Módulo 1

Tela principal do sistema. Carregada automaticamente ao abrir o app, exibe um resumo completo da situação financeira do mês atual com indicadores visuais, gráficos interativos e as transações mais recentes.

  • 4 cards de KPI: Saldo do Mês, Total Receitas, Total Despesas e Taxa de Economia
  • Gráfico de pizza (Chart.js) com despesas agrupadas por categoria
  • Gráfico de barras com comparativo de receitas vs despesas nos últimos 6 meses
  • Tabela das 8 transações mais recentes do mês com tipo, categoria e valor
  • Indicador de mês atual no cabeçalho do card de gráfico
  • Taxa de Economia calculada: (receitas - despesas) / receitas × 100
  • Cores dinâmicas: verde para valores positivos, vermelho para negativos
  • Gráficos redimensionados automaticamente ao trocar tema claro/escuro
💳
Financeiro
Gerenciamento completo de receitas e despesas
Módulo 2

Central de controle financeiro. Permite adicionar, editar e excluir transações, além de filtrar por período, tipo e categoria com resumo financeiro atualizado em tempo real.

  • Filtros combinados: mês/ano, tipo (receita/despesa) e categoria
  • Cards de resumo: Saldo, Total Receitas e Total Despesas do filtro atual
  • Tabela completa com data, descrição, categoria, tipo e valor de cada transação
  • Botões de editar (reabre o modal preenchido) e excluir para cada linha
  • Modal de nova transação com seletor de tipo, valor, data, categoria e descrição
  • 10 categorias de receita e 10 categorias de despesa pré-definidas
  • Contador de transações exibindo quantas foram encontradas com os filtros aplicados
  • Estado vazio visual com ícone e mensagem quando nenhuma transação é encontrada
📅
Agenda
Calendário interativo com gestão de eventos
Módulo 3

Agenda visual com calendário mensal completo. Permite registrar compromissos, reuniões, lembretes e tarefas com horário, tipo e status de execução.

  • Calendário mensal com dias, headers de semana e navegação entre meses
  • Indicador visual (ponto colorido) nos dias que possuem eventos cadastrados
  • Destaque automático do dia atual no calendário
  • Seleção de dia: clique em qualquer dia para ver apenas os eventos daquele dia
  • Lista de eventos com horário, tipo (badge colorido) e status
  • Filtro por status: pendente, concluído ou cancelado
  • Modal de evento: título, data, horário, tipo, status e descrição
  • 4 tipos de evento: Evento, Reunião, Lembrete, Tarefa
  • 3 status: Pendente, Concluído, Cancelado — cada um com badge visual distinto
📝
Anotações Pessoais
Bloco de notas colorido e personalizável
Módulo 4

Módulo de anotações livres com visual de cartões coloridos. Ideal para registrar metas financeiras, lembretes, ideias e observações importantes.

  • Grade de cartões responsiva com mínimo de 260px por cartão
  • 8 opções de cor para personalizar cada nota (índigo, verde, vermelho, amarelo, azul, rosa, roxo, verde-água)
  • Cada cartão exibe título, prévia do conteúdo e data da última atualização
  • Barra lateral colorida em cada cartão refletindo a cor escolhida
  • Modal de edição com título, conteúdo (textarea) e seletor de cores visual
  • Escape de HTML no conteúdo para segurança (evita XSS)
  • Estado vazio com mensagem de encorajamento ao criar a primeira nota
  • Ordenação por data de atualização (mais recentes primeiro)
📈
Relatórios
Análise anual detalhada com gráficos e tabelas
Módulo 5

Central de inteligência financeira. Consolida todos os dados do ano selecionado em gráficos analíticos e uma tabela de participação por categoria, permitindo identificar padrões de consumo.

  • Seletor de ano (últimos 6 anos disponíveis)
  • 4 cards anuais: Total Receitas, Total Despesas, Saldo do Ano e Taxa de Poupança
  • Gráfico de linhas (Chart.js) com receitas e despesas mês a mês ao longo do ano
  • Gráfico de rosca (doughnut) com distribuição de despesas por categoria
  • Tabela de categorias: nome, tipo, valor total, quantidade de transações e participação %
  • Barra de progresso visual mostrando a participação percentual de cada categoria
  • Média mensal de despesas calculada automaticamente
  • Dados agregados com queries SQL no backend (sem processamento pesado no JS)
⚙️
Configurações
Personalização e segurança dos dados
Módulo 6

Painel de controle do usuário. Permite personalizar configurações pessoais, organizar dados vitais e garantir a segurança das informações através de backups offline.

  • Perfil do Usuário: Salvar nome completo, CPF e instruções pessoais (armazenamento local)
  • Categorias Personalizadas: Adicione e gerencie novas categorias próprias para receitas e despesas
  • Integração total: As categorias refletem instantaneamente no seletor de novas transações
  • Exportação de Backup: Baixe um backup em formato JSON (transações, agenda, notas e categorias)
  • Restauração: Carregue um backup baixado anteriormente para recuperar o estado do sistema
  • Zona de Perigo: Botão dedidaco para limpar de forma irreversível os dados da sessão/banco
07 — Deploy

Processo de Deploy na Vercel

O FinanceFlow está disponível em produção através de um pipeline de deploy automatizado: GitHub → Vercel → Neon. Qualquer novo git push na branch main dispara um novo deploy automaticamente.

Banco de Dados — Neon.tech

Criação de projeto PostgreSQL gratuito no Neon. Copia-se a Connection String no formato postgres://user:senha@host.neon.tech/financeflow?sslmode=require.

Repositório — GitHub

Código enviado ao GitHub com git init → git add → git commit → git push. Repositório público ou privado. Arquivo .env jamais é commitado (está no .gitignore).

Projeto — Vercel

Conectado ao repositório GitHub via vercel.com. Framework detectado automaticamente como Other (sem build step necessário).

Variável de Ambiente

A Connection String do Neon é adicionada no painel da Vercel como variável de ambiente DATABASE_URL. Isso mantém a senha segura, fora do código.

Deploy Automático

A Vercel faz o deploy e fornece uma URL pública (finance-flow.vercel.app). Na primeira requisição, o initDB() cria todas as tabelas automaticamente no Neon.

Custo total de infraestrutura: R$ 0,00. Vercel Hobby (gratuito) + Neon Free Tier (gratuito) + GitHub (gratuito) = sistema em produção sem nenhum custo.

Banner nativo exibido automaticamente no celular quando o critério de instalação é atendido (HTTPS + manifest + SW).

🎨

A cor da barra de status do sistema operacional muda conforme o tema (escuro/claro) selecionado pelo usuário.

09 — Segurança

Práticas de Segurança

AmeaçaProteção Implementada
SQL Injection Queries parametrizadas ($1, $2...) em todos os endpoints. Nenhum valor do usuário é interpolado diretamente no SQL.
XSS (Cross-Site Scripting) Função escapeHtml() aplicada em todo conteúdo de notas antes de renderizar no DOM.
Exposição de credenciais DATABASE_URL armazenado apenas como variável de ambiente na Vercel. O arquivo .env está no .gitignore.
Requisições indevidas Cada endpoint valida campos obrigatórios e retorna erro 400 com mensagem específica se os dados estiverem incompletos.
Método HTTP incorreto Todos os endpoints retornam erro 405 para métodos não suportados.
CORS Headers CORS configurados nos serverless functions para permitir chamadas do frontend. Configuração adicional no vercel.json.
